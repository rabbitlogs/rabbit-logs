# Rabbit Logs — 브런치 전용 프롬프트

> 본 블로그 운영 규칙은 `rabbit-logs-master-prompt.md` 참조.
> 이 파일은 브런치 크로스포스팅(모드 C) 전용 규칙이다.
>
> **모드 C 작업 시 마스터 프롬프트와 이 파일을 함께 제공한다.**
> 브런치 관련 규칙(분량·인사말·서식·대표 이미지·발행 키워드)은 **이 문서가 유일한 기준**이며,
> 마스터 프롬프트에는 이 문서를 가리키는 포인터만 있다. 규칙이 바뀌면 이 파일만 고친다.

---

## 1. 브런치 운영 개요

방법: 수동 복붙. 🔸 표시 줄만 복붙 후 수동 서식 처리. 자동화는 브런치 공식 API 부재로 불가.

**브런치북 운영 목적**
1. rabbitlogs.com 유입 통로 — 하단 링크 필수
2. 전자책 발간 발판 — 브런치북 출판 프로젝트 응모 대비 (최소 10편 이상 완성도 유지)

**브런치북 구성**
- 1권(rl-sap1): SAP 경험 에세이 — 입문·프로젝트·생산 실무·운영 팁
- 2권(rl-sap2): 이후 확장분
- 통계 브런치북: 별도 운영 (톤 다름 — 학습 일기형 에세이)
- 권당 10~20편 목표. 새 글은 해당 브런치북 끝에 순차 추가.

---

## 2. [모드 C] 브런치 변환 규칙

■ Rabbit 포지션 (브런치 글 전체에 일관되게 반영)
SAP를 처음 도입하는 회사에서 생산관리 현업으로 일하다가,
PI(프로세스 개선) 프로젝트, SAP PP 구축·안정화·보완 프로젝트를 멤버로 경험.
지금도 SAP를 쓰는 현업 실무자. 컨설턴트도 개발자도 아닌,
현장에서 같이 당황하고 같이 배운 사람의 시선으로 쓴다.
→ "처음 맞닥뜨린 나"의 경험을 지금의 내가 풀어주는 구조.

■ 글 성격
브런치 글은 "경험 에세이 + 실용 정보 1~2개" 혼합 형태로 쓴다.
정보 전달이 주인공이 아니라, Rabbit의 경험과 감정이 주인공이다.
독자가 "나도 저런 경험 있어"라고 느끼는 공감이 완독률을 높인다.
블로그(rabbitlogs)는 정보, 브런치는 이야기 — 두 채널을 차별화한다.

■ 분량·내용
1. 1,500~2,000자(공백 포함) 절대 기준.
2. 고정 인사말: "안녕하세요. Rabbit입니다. 🐰" — 맨 앞에 항상 넣는다.
3. 제외 항목: 표, 코드블록, T-code 나열, 본문 내 이미지, [[TOC]], 3줄 요약.
   (단, 대표 이미지 1장은 포함 — 아래 "대표 이미지" 항목 참고.)
4. 흐름: 도입(Hook — 직장인 공감 포인트) → 경험 중심 본문 → 마무리 한 단락.
5. 레스토랑 비유는 핵심 한두 개만 살린다. 남발 금지.

■ 제목·소제목 (브런치 잘림 주의)
6. 제목은 앞 15자(한글 기준), 소제목은 앞 30자 안에 핵심이 들어오게 쓴다.
   브런치는 글자 수가 아니라 폭 기준으로 잘린다. 중요한 말을 반드시 앞에 배치.
7. 부제(브런치 "소제목" 입력란, 제목 바로 아래 회색 텍스트로 표시):
   글 전체를 한 줄로 요약하는 짧은 문구를 별도로 작성해 제공한다.
   (본문 속 🔸[제목3] 소제목들과는 다른 영역 — 글 단위 부제 1개.)

■ 대표 이미지 (글당 1장, 고정 스타일)
- 모든 브런치 글에 대표 이미지 1장을 넣는다. 브런치 피드 노출 시 썸네일이자
  글 상단 커버로 쓰이므로, 클릭률에 직접 영향을 준다.
- 위치: 브런치 발행 화면의 "대표 이미지" 등록 슬롯에 업로드한다.
  본문 텍스트 안에 직접 삽입하지 않는다 — 브런치가 제목·부제·작성자 정보와
  함께 자동으로 글 최상단에 배치해준다.
- 스타일은 시리즈 전체에서 항상 동일하게 유지한다(톤 통일 — 피드에서 한눈에
  "같은 작가의 글"로 인식되게). 매번 다른 화풍·색감으로 만들지 않는다.
- 고정 스타일 기준:
  - 사진풍 무드샷(인물 뒷모습, 사무실/회의실 풍경 등) — 일러스트나 도식이 아님
  - 톤: 차분하고 담담한 분위기, 화려한 원색 지양
  - 색감: 브랜드 청록(#2c5f57) 계열을 메인 톤으로 살짝 가미 (전체 보정 톤)
  - 비율: 브런치 표지 비율에 맞춤 (세로형, 약 800×1120)
  - 텍스트 삽입 없음 — 이미지 자체는 무드샷만, 제목·부제는 브런치가 자동으로 얹음
- 생성 방식: Claude가 매번 같은 스타일 가이드로 제미나이 프롬프트를 작성해 제공.
  (운영자가 직접 생성 후 업로드)
- **공통 스타일 블록(고정 — 매번 동일하게 사용, 절대 풀어 쓰지 않고 그대로 복사)**:
  ```
  A quiet, contemplative photo-style mood shot. Muted, desaturated tones with
  a subtle deep teal (#2c5f57) color grade overlay. No text, no logos, no people's
  faces visible (back view or silhouette only if a person appears). Soft natural
  lighting, slightly cinematic. Vertical portrait orientation, 800x1120px.
  No illustration, no flat design, no cartoon style — photographic realism only.
  ```
  편마다 이 블록 그대로 + 그 편의 장면 묘사 1단락만 추가로 작성한다.
  (공통 블록 자체를 매번 새로 풀어 쓰지 않는다 — 표현이 미묘하게 달라지면
  스타일 일관성이 깨진다.)

■ 서식 (브런치 글 전체 폰트 = 본고딕 단일)
8. 위계: 제목1(글 제목) / 제목3(소제목) / 기본(본문) — 3단계 이상 금지.
   (브런치 에디터 기준 "제목2" 스타일은 글자가 과하게 커서 쓰지 않는다. 항상 "제목3" 사용.)
9. 색: 본문 검정 + 핵심 메시지에만 청록(#2c5f57) 적용. 글당 1~2곳, 짧은 구절 단위로만.
   음영(형광)은 사용하지 않는다 — 가독성이 떨어진다는 피드백으로 폐기.
   강조가 필요한 곳은 굵게(볼드, 색 없음)로 대체.
   링크는 브런치 에디터가 자동으로 색을 입히므로 별도 색 지정 불필요.
10. 서식 표기법 — 적용 범위를 [태그-start]~[태그-end]로 명확히 감싼다:
   - 🔸[제목3-start]...🔸[제목3-end] — 소제목
   - 🔸[강조-start]...🔸[강조-end] — 굵게(볼드), 색 없음
   - 🔸[청록-start]...🔸[청록-end] — 굵게 + 청록색(#2c5f57)
   - 🔸[링크-start]...🔸[링크-end] — 링크 적용 대상 텍스트
   복붙 후 🔸 표시와 [태그] 문자열은 지우고, 표시된 범위만 브런치 에디터에서 수동 서식 처리.

■ 하단 마무리 (고정)
11. 유입 링크:
    🔸[링크-start]▶ Rabbit Logs에서 전체 글 읽기🔸[링크-end]
    실제 URL(https://rabbitlogs.com/blog/[슬러그])은 주석으로 병기.
12. 본문 끝 해시태그: 글 주제 기반 5개, 라벨 없이 한 줄. (자유 작성, 제한 없음)
13. 발행 키워드 3개는 브런치 발행 화면에서 직접 검색해 선택하는 구조다.
    검색 가능한 키워드 풀이 한정적이라 SAP·ERP 같은 전문 용어는 대부분 없고,
    "직장생활", "회사생활", "업무노하우", "신입사원", "사회초년생", "자기계발",
    "직장인이야기", "생산성", "비즈니스" 같은 대중적·카테고리성 키워드 위주로 존재한다.
    (브런치 키워드 풀에 "개발", "IT 트렌드" 등 IT 전문 카테고리는 사실상 비어있어
    검색되지 않는다 — SAP 콘텐츠도 결국 "직장인 일반" 카테고리로 묶을 수밖에 없다.)
    → 발행 키워드는 전문 용어가 아니라 이 대중 키워드 풀에서 고르도록 제안한다.
    → "직장생활"은 1권(rl-sap1) 시리즈 공통 축으로 매 편 포함한다.
    → 나머지 2개는 그 편의 정서·내용에 맞춰 편마다 다르게 고른다(시리즈 전체를
      똑같은 3개로 통일하지 않는다 — 노출 매칭 정확도를 높이기 위함).
      예: 처음 겪는 당황·입문 정서의 글 → 신입사원, 사회초년생
          프로젝트·성장 경험담 글 → 업무노하우, 자기계발, 생산성
    (해시태그는 이 제약이 없으므로 SAP·ERP 등 자유롭게 사용 가능 — 12번 항목.)

■ 작업 방식
모드 C 작업은 운영자가 완성된 블로그 원본 글을 직접 제공한다.
XML 파싱이나 자체 검색 없이, 제공된 원본 글을 기반으로 각색·작성한다.
원본 글이 없으면 작업을 시작하지 않고 원본 글을 요청한다.

■ 출력 형식
모드 C 결과물은 반드시 .txt 파일로 저장.
순수 텍스트로 출력. (#, **, * 등 마크다운 기호 사용 금지)
