# Rabbit Logs — 사이트 구현 사양

> 마스터 프롬프트(`rabbit-logs-master-prompt.md`)에서 분리한 문서.
> 여기 있는 것은 **이미 구현이 끝난 사이트 사양**이라 글을 쓸 때마다 참고할 필요가 없다.
> 사이트 구조를 바꾸거나 새 카테고리를 추가할 때만 열어 본다.
>
> 마스터 프롬프트에는 "글과 이미지를 만드는 규칙"만 남긴다.
>
> 섹션 번호는 이 문서 자체 기준이다(분리 전 마스터의 16·17·18·21에 해당).

## 1. /map 페이지 & 글 하단 미니맵 (구현 확정)

### /map 페이지
- **인터랙션**: 마우스 오버 = 연결 강조(관련선도 그때만 표시), 클릭 = 즉시 글 이동(더블클릭 아님).
  모바일(터치)은 1차 터치 = 강조, 같은 노드 재터치 = 이동, 빈 화면 터치 = 강조 해제.
- **화살표 없음** — markerUnits/줌 스케일 이슈로 안정적으로 안 보여 완전히 제거.
  방향성은 범례 설명("선행: 먼저 읽기" 등)과 레이아웃으로만 전달.
- **관련(related) 관계선은 기본 숨김** — 평소엔 선행/심화만 보이고, 노드 강조 시에만 관련선이 같이 뜸
  (안 그러면 28개 글 + 태그가 거미줄처럼 얽혀 보임).
- **카테고리 필터 시 태그도 함께 필터링** — 선택한 카테고리 글과 실제 연결된 태그만 표시.
- **태그 토글 기본 OFF** — "+ 태그" 버튼으로 켜고 끔. 색상은 보라/모브(#9b7fa6) 계열.
- 새 카테고리 추가 시 `catColor` 객체와 카테고리 필터 칩에도 항목 추가 필요.

### 글 하단 1-hop 미니맵
- 위치: 본문("더 읽어보기" 포함) 바로 아래, 태그 영역 위.
- **태그 노드는 표시 안 함** — 글 관계(선행/관련/심화)만, 화살표도 없음.
- 양방향 관계 모두 포함 — 이 글이 가리키는 것 + 이 글을 가리키는 다른 글 둘 다
  (한쪽만 보면 certification처럼 관계의 "받는 쪽"인 루트 글에서 미니맵이 안 뜨는 버그가 났었음).
- 범례는 그래프 **아래**, 설명 문구 없이 "선행/관련/심화"만.
- 우측 상단에 "전체 지도 보기 →" 링크로 `/map` 연결.
- 서버사이드 정적 SVG로 구현(JS 없음) — 가볍고 접근성·SEO에 안전.

## 2. 헤더 구조 (확정)

- 지도 아이콘(네트워크 모양 SVG) — 검색 아이콘 바로 왼쪽, `/map` 링크.
- 카테고리 메뉴는 **항상 둘째 줄**(데스크탑 포함) — 지도·검색·언어토글과 같은 줄에 두면 답답해 분리.
- **카테고리 메뉴 오른쪽 정렬** (`justify-content: flex-end`, PC 기준).
- **로고~카테고리 구분선 없음** (`border-top` 제거).
- 모바일에서는 카테고리 라벨에서 "SAP" 접두어 생략(짧은 라벨 별도 정의) — 좁은 화면 한 줄에 다 들어가게.
- **모바일 카테고리 소개글 줄바꿈 없음** (`cat-desc-br` 클래스로 모바일에서 숨김).
- 카테고리 순서: 프로젝트 → 실무운영 → 학습자료 → 통계·데이터 → 소개&문의.

## 3. 홈 / About (확정)

레스토랑 비유는 **블로그 글 본문 전용**이다. 홈페이지·About처럼 브랜드 정체성을 보여주는 페이지는
비유 없이 "코드는 몰라도, 프로세스는 잘 아는 사람"이라는 실무자 포지셔닝으로 직접 표현한다.
새 카테고리(stats) 소개 문구도 이 원칙을 따른다 — 비유 없이 담백하게.

**홈 히어로 서브**: "생산 PI·SAP PP 실무 기록, 그리고 데이터로 넓혀가는 과정."

**About 최종 확정 카피**:
```
문단 1: 생산 PI와 SAP PP 구축, 안정화 프로젝트를 거쳐,
        지금도 SAP를 '사용하는' 현업 실무자로 일하고 있습니다.

문단 2: 일하면서 배운 것을, 일하는 사람의 언어로 기록합니다.
        그리고 데이터로 시야를 넓혀가는 과정도 함께 남깁니다.

모토(중앙 정렬 유지): 기능보다 프로세스, 화면보다 사람.
```
(경력=누구인지 → 기록 이유+데이터 방향 순서가 더 자연스러운 흐름.)

**카테고리 소개 문구**(2문장, 마침표 기준 줄바꿈)는 `src/pages/category/[cat].astro`의
`catDesc` 객체에서 관리. 새 카테고리 추가 시 이 객체에 항목 추가 필수.

## 4. 슬러그(URL) 변경 시 리다이렉트 규칙

워드프레스 원본에서 슬러그가 바뀌어 새로 발행하거나, 리빌딩 중 슬러그를 바꾸면
`public/_redirects`에 반드시 한 쌍(한글 원본 + URL인코딩본)을 추가한다:

```
/예전-한글-슬러그/   /blog/새-영문-슬러그   301
/sap-%ec%98%88%ec%a0%84-%ec%9d%b8%ec%bd%94%eb%94%a9/   /blog/새-영문-슬러그   301
```

원본 워드프레스 구조는 `rabbitlogs.com/{슬러그}/` 평면 구조였고, 새 Astro 블로그는
`rabbitlogs.com/blog/{슬러그}/` 구조라 **슬러그가 같아도(`sap-mrp` 등) 경로 prefix가
바뀌었으므로 리다이렉트가 필요하다.**
