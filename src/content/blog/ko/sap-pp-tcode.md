안녕하세요, Rabbit입니다! 🐰

SAP PP(생산관리) 실무자라면 하루에도 수십 번씩 T-CODE를 마주합니다. "CO03으로 오더 확인하고, MD04로 소요량 보고, COOIS로 필터 걸고, VL02N으로 출하 수정하고…" 이런 흐름이 매일 반복되죠.

생산관리는 수주부터 생산, 출하까지 전 과정을 아우르다 보니 찍어야 할 코드도, 확인할 정보도 많습니다. 그래서 오늘은 PP 실무에서 자주 쓰는 T-CODE를 **업무 흐름순**으로 정리했습니다. "그 코드 뭐였더라?" 싶을 때 빠르게 찾아보는 용도로 두시면 좋습니다.

> **참고**: 실무에서 새로 생성(Create)하는 작업은 대부분 회사가 만든 Z코드를 쓰고, 표준 T-CODE는 오히려 변경(Change)·조회(Display)에 더 자주 쓰입니다. 그래서 이 글도 조회·변경 중심으로 구성했습니다. T-CODE 입력법이 낯설다면 [SAP T-CODE, 주방 단골 메뉴를 빠르게 부르는 법](/blog/sap-tcode-basics)을 먼저 보세요.

## 1. 주문 (Sales Order)

생산의 시작은 고객 주문입니다.

| T-CODE | 설명 | 주요 기능 |
|---|---|---|
| VA01 | 판매 오더 생성 | 고객 주문 생성 |
| VA02 | 판매 오더 변경 | 수량·납기 등 변경 |
| VA03 | 판매 오더 조회 | 오더 상태 및 상세 확인 |

*표 1. 주문(Sales Order) 단계 주요 T-CODE*

## 2. 생산 계획 및 오더 (Production)

계획이 있어야 생산이 돌아갑니다. PP 실무에서 손이 가장 많이 가는 코드들입니다.

| T-CODE | 설명 | 주요 기능 |
|---|---|---|
| MD01N | MRP Live | 전사 자재 소요계획 실행 |
| MD04 | 재고/소요 현황 | 자재별 수급 현황 확인 |
| MD16 | 계획오더 일괄조회 | 전체 계획오더 리스트 |
| CO01 | 생산오더 생성 | 생산지시서 생성 |
| CO02 | 생산오더 변경 | 수량·날짜 등 변경 |
| CO03 | 생산오더 조회 | 생산오더 상세 확인 |
| COHV | 생산오더 대량처리 | 생산오더 일괄 처리 |
| COOIS | 생산오더 정보시스템 | 오더 리스트 조회·필터링 |
| CA01 / CA02 / CA03 | 공정(Routing) 생성·변경·조회 | 공정 경로 관리 |
| CS01 / CS02 / CS03 | BOM 생성·변경·조회 | 자재 명세서 관리 |
| CS12 | 다중 레벨 BOM 조회 | 하위 BOM까지 전체 구조 |
| CS15 | 사용처 리스트 | 해당 자재가 쓰인 상위 BOM |
| C223 | 생산 버전 유지보수 | 생산 버전 등록·관리 |

*표 2. 생산 계획·오더(Production) 단계 주요 T-CODE*

## 3. 자재 관리 및 구매 (Material)

자재가 있어야 생산이 가능합니다. 입고·재고·구매 확인에 쓰는 코드들입니다.

| T-CODE | 설명 | 주요 기능 |
|---|---|---|
| MM01 / MM02 / MM03 | 자재 마스터 생성·변경·조회 | 자재 마스터 데이터 |
| MM17 | 자재마스터 대량 유지보수 | 여러 자재 일괄 수정 |
| MMBE | 재고 개요 | 현재·예약 재고 현황 |
| MB52 | 창고별 재고 리스트 | 저장위치별 수량 확인 |
| MB51 | 자재 문서 리스트 | 입출고 히스토리 조회 |
| MIGO | 자재 이동 | 입고·출고·이동 처리 |
| ME52N | 구매 요청(PR) 변경 | 생성된 PR 수정 |
| ME21N | 구매 오더(PO) 생성 | 공급사 PO 생성 |
| ME13 | 구매 정보 레코드 조회 | 공급업체별 단가·조건 |
| MB5S | GR/IR 잔고 조회 | 입고·송장 불일치 확인 |
| MSC3N | 배치(Batch) 조회 | 자재별 LOT 이력 확인 |

*표 3. 자재 관리·구매(Material) 단계 주요 T-CODE*

## 4. 출하 및 물류 (Delivery)

생산품이 제대로 나가는 것도 중요합니다. 출하·선적·운송비 처리 코드들입니다.

| T-CODE | 설명 | 주요 기능 |
|---|---|---|
| VL01N | 출하 생성 | 납품 문서 생성 |
| VL02N | 출하 변경 | 납품 문서 수정 |
| VL03N | 출하 문서 조회 | 납품 전표 상세 확인 |
| VL09 | 출고 취소 | 잘못 출고된 전표 되돌리기 |
| VT01N / VT02N / VT03N | 선적 생성·변경·조회 | 선적 문서 관리 |
| VI01 / VI02 / VI03 | 선적비용 생성·변경·조회 | 운송비·보험료 전표 |

*표 4. 출하·물류(Delivery) 단계 주요 T-CODE*

## Z코드도 잊지 마세요

표준 T-CODE 외에, 각 회사가 자체 개발한 **Z코드**가 있습니다. 이름 그대로 알파벳 Z로 시작하죠. 예를 들어 `ZBOMPRINT`(BOM 출력)나 `ZMM_REPORT`(자재 맞춤 리포트) 같은 것들입니다. 회사 업무에 딱 맞춰 만든 거라, 실무에서는 표준 코드보다 더 자주 쓰이기도 합니다. 표준 코드 목록과 별개로, 우리 회사의 Z코드도 따로 정리해두시길 권합니다.

## Rabbit의 한 끗

생산관리는 수주부터 출하까지 모든 흐름이 연결돼 있습니다. 그래서 T-CODE도 **흐름 단위로 묶어서** 익히는 게 효율적입니다.

예를 들어 주문(VA01) → 생산오더(CO01) → 재고 확인(MMBE) → 출하(VL01N)처럼 자주 이어지는 묶음을 즐겨찾기에 등록해두면, 화면을 오가지 않고 작업 속도가 눈에 띄게 빨라집니다. 생성은 Z코드로 처리하더라도, 오류 확인이나 일정 조정은 표준 코드로 대응할 일이 많으니 CO03·MD04·COOIS 같은 조회 코드는 손에 익혀두길 권합니다.
