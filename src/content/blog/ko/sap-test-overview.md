안녕하세요, Rabbit입니다! 🐰

수십억을 들이고 수많은 사람이 1–2년 매달린 SAP 시스템이 드디어 오픈하는 날. 그런데 그날 영업팀에서 "주문이 안 들어가요", 생산팀에서 "잘못된 계획이 내려와 라인이 멈췄어요", 회계팀에서 "매출이 하나도 안 맞아요"라는 말이 동시에 터진다면 어떨까요?

상상만 해도 아찔한 이 대참사를 막는 과정이 바로 **테스트**입니다. SAP 프로젝트에서 "대충 만들고 일단 오픈하자"는 통하지 않습니다. "믿되, 반드시 검증한다"가 원칙이죠. 오늘은 그 검증 과정인 테스트 5단계를, 새 레스토랑 매장을 여는 과정에 빗대어 전체 그림을 잡아보겠습니다.

> **3줄 요약**
> - SAP 테스트는 오픈 후의 대참사를 막기 위한 필수 검증 과정이다.
> - Fit/Gap → 프로토타입 → 단위 → 통합 → UAT, 5단계로 흐른다.
> - 건너뛰거나 순서를 바꿀 수 없는, 긴밀하게 연결된 여정이다.

## 왜 테스트에 이렇게 공을 들일까

"잘 만들면 됐지, 테스트를 왜 이렇게 복잡하게 하나" 싶을 수 있습니다. 이유는 분명합니다.

첫째, 위험을 줄입니다. 시스템 오류로 인한 업무 중단이나 금전 손실 같은 위험을 미리 막습니다. 둘째, 품질을 보증합니다. 만들기로 약속한 기능이 정확히 작동하는지 확인합니다. 셋째, 신뢰를 확보합니다. 프로젝트팀에는 "믿고 내놔도 된다"는 자신감을, 현업에는 "믿고 써도 된다"는 신뢰를 줍니다.

새 매장도 마찬가지입니다. 주방 설비만 들여놓고 바로 손님을 받지 않습니다. 가스는 잘 들어오는지, 동선은 막히지 않는지, 포스기와 주방이 연동되는지 차근차근 점검한 뒤에야 문을 엽니다.

## 새 매장을 여는 5단계 점검

새 매장이 콘셉트 확정, 시범 운영, 재료 검수, 주방·홀 연동, 최종 점검을 거쳐 문을 열듯, SAP 테스트도 5단계로 이어집니다.

![Fit/Gap, 프로토타입, 단위, 통합, UAT로 이어지는 SAP 테스트 5단계 흐름도](/images/sap-test-flow.jpg)
*그림 1. SAP 테스트 5단계 — 새 매장을 여는 검증의 여정*

### 1단계: Fit/Gap 분석 — 콘셉트가 우리 매장에 맞는가

본사 표준 메뉴와 운영 방식이 우리 매장에 잘 맞는지 따져보는 단계입니다. 우리가 원하는 미래의 업무 방식(To-Be)이 SAP 표준 기능과 얼마나 맞는지(Fit), 달라서 따로 만들거나 고쳐야 할 부분은 무엇인지(Gap)를 가려냅니다. 설계 단계의 최종 검토인 셈입니다.

### 2단계: 프로토타입 테스트 — 시범 매장을 먼저 본다

본격적인 공사 전에 시범 매장을 차려 미리 보여주는 단계입니다. "동선은 편한지, 화면은 쓰기 좋은지"를 사용자가 직접 체험하게 하죠. Fit/Gap에서 정한 개발 항목을 시제품으로 만들어 미리 검증하고, 추가 요구사항을 반영하는 '미리보기' 단계입니다.

### 3단계: 단위 테스트 — 재료 하나하나의 품질 검사

매장에 들인 식자재를 하나하나 꺼내 신선도와 상태를 검사하는 과정입니다. 개발자나 컨설턴트가 자신이 만든 가장 작은 단위의 기능(프로그램, 함수)이 독립적으로 완벽히 작동하는지 스스로 검증하는 '부품 품질 검사' 단계입니다.

### 4단계: 통합 테스트 — 주방과 홀이 제대로 연동되는가

검증된 재료로 주방을 차리고, 홀과 주방을 연결합니다. "홀에서 주문을 넣으면 주방 모니터에 제대로 뜨는가"를 확인하는 단계죠. 영업·구매·생산·회계 같은 서로 다른 모듈이 데이터를 주고받으며, 하나의 업무 프로세스가 처음부터 끝까지 막힘없이 흐르는지 검증하는 '연동성 테스트'입니다.

### 5단계: UAT — 문 열기 전, 사장의 최종 점검

드디어 매장이 완성됐습니다. 실제로 일할 직원(현업 사용자)을 불러, "주문은 잘 들어가는지, 결제는 매끄러운지" 직접 써보게 하고 최종 승인을 받는 과정입니다. 실사용자 관점에서 "이 시스템이 우리 업무에 정말 적합한가"를 확인하고 승인하는 마지막 테스트입니다.

## Rabbit의 한 끗

이 다섯 단계는 건너뛰거나 순서를 바꿀 수 없는, 서로 긴밀하게 연결된 여정입니다. 콘셉트 검토(Fit/Gap) 없이 시범 매장(프로토타입)을 차릴 수 없고, 재료 검사(단위) 없이 주방 연동(통합)을 확인할 수 없으니까요.

다음 글부터 각 단계를 하나씩 자세히 들여다보겠습니다. 이 개요를 머릿속에 지도처럼 두고 따라오면, 복잡해 보이던 SAP 테스트도 새 매장을 여는 자연스러운 준비 과정으로 보일 거예요. 😎
